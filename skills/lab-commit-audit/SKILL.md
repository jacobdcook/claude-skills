# Lab commit audit

Jacob wants commits to look human and **never** advertise Cursor, Claude, or other AI tools. Aligns with project rule: no `Co-authored-by:` AI lines and no AI attribution in commit messages.

---

## When to run

- After **`lab-daily` full delivery** (build + commit + push).
- Before sharing a branch or merging a PR.
- When the user says **audit commit**, **lab audit**, **`/audit-commit`**, **check my last commits for AI**, or similar.

---

## Step 1 — Inspect recent commits

From the relevant repo:

```bash
git log -20 --format=fuller
```

---

## Step 2 — Flag patterns (case-insensitive)

**FAIL** (report and suggest fix):

| Signal | Why |
|--------|-----|
| `Co-authored-by:` with **Cursor**, **Claude**, **Anthropic**, **OpenAI**, **Copilot**, **ChatGPT**, **aider**, **Codex**, **Gemini**, or similar | AI attribution |
| `Generated by`, `🤖`, `via Claude`, `via Cursor`, `AI-generated` in subject or body | Same |
| Subject empty or junk only: `.`, `wip`, `fix` (alone), `update` (alone), `asdf` | Ping-style commit |

**WARN:** author ≠ committer (often benign), or oddly long boilerplate body.

**PASS:** Clean subject, no AI trailers, Jacob’s usual author identity.

---

## Step 3 — Output

For each commit: SHA, subject, **PASS/FAIL/WARN**, and if FAIL the offending line.

**Last-commit fix:**

```bash
mkdir -p /tmp/nohooks
git commit --amend -m "Clean subject" -c core.hooksPath=/tmp/nohooks
```

If already pushed: note **force-with-lease** only with explicit user consent on safe branches.

---

## Step 4 — Agents: prevent dirty commits

```bash
mkdir -p /tmp/nohooks
git -c core.hooksPath=/tmp/nohooks commit -m "Short human message"
```

No `Co-authored-by:`. No AI tool names in the message.
