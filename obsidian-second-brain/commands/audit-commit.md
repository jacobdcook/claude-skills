# /audit-commit — Audit recent git commits for AI trailers and junk

Optional arg: a repo path or repo name (e.g. `/audit-commit network-behavior-analyzer`). If omitted, audit the current working directory's repo.

## Do this

1. **Resolve repo path:**
   - If arg looks like an absolute path or starts with `~`, use it directly.
   - If arg is a bare name, try `~/Desktop/PROJECTS/<name>` first, then `~/<name>`.
   - If no arg, use `git rev-parse --show-toplevel` from cwd.
   - If the directory is not a git repo, stop and tell the user.

2. **Inspect recent commits** (last 20):

   ```bash
   git -C <repo> log -20 --format=fuller
   ```

3. **Flag patterns (case-insensitive)** against each commit's subject + body + trailers:

   **FAIL** signals:
   - `Co-authored-by:` containing **Cursor**, **Claude**, **Anthropic**, **OpenAI**, **Copilot**, **ChatGPT**, **aider**, **Codex**, **Gemini**, **Windsurf**, **Cline**, **Devin**, or similar AI tool/vendor names.
   - `Generated by`, `🤖`, `via Claude`, `via Cursor`, `AI-generated`, `Generated with` anywhere in the message.
   - Subject empty or junk-only: `.`, `wip`, `fix` (alone), `update` (alone), `asdf`, `test` (alone).

   **WARN** signals:
   - Author ≠ Committer (often benign — note it, don't fail).
   - Oddly long boilerplate body that looks templated.
   - Multiple consecutive commits with identical subjects.

   **PASS:** Clean subject, no AI trailers, Jacob's usual author identity (`jacobdcook`).

4. **Output format** — for each of the recent commits:

   ```
   <short-sha> <PASS|FAIL|WARN> <subject>
       └─ <reason line, only if FAIL or WARN>
   ```

   Then a summary line: `N commits audited — X PASS, Y WARN, Z FAIL`.

5. **If any FAIL on last commit (unpushed):**

   Offer this exact fix (do not run without confirmation):

   ```bash
   mkdir -p /tmp/nohooks
   git -C <repo> -c core.hooksPath=/tmp/nohooks commit --amend -m "Clean subject here"
   ```

6. **If any FAIL on already-pushed commits:**

   Note this requires history rewrite. Do NOT run force-push without explicit user consent. Mention `git push --force-with-lease` as the safest option, only on branches Jacob controls (not shared/main on team repos — `jacobdcook` solo repos are fine).

7. **Codex second opinion (optional, only if user adds `--codex` or asks for it):**

   Run codex non-interactively to cross-check the audit:

   ```bash
   git -C <repo> log -20 --format=fuller | codex exec --skip-git-repo-check "Audit these git commits. Flag any that contain AI tool attribution (Cursor, Claude, Anthropic, OpenAI, Copilot, ChatGPT, Codex, Gemini, etc.), Co-authored-by lines pointing to AI tools, 🤖 emojis, 'Generated by' phrases, or empty/junk subjects. For each flagged commit return: SHA, the offending line, and a one-line fix suggestion. Be terse."
   ```

   Display codex's verdict alongside the local audit. If they disagree, surface that explicitly.

## Why this skill exists

Jacob's repos are public portfolio pieces. AI attribution in commits signals "this person used a tool to do their work" to recruiters and hiring managers. The mission is for commits to read as human-authored.
